UX Design:

CollegeVibes





Summary of the Application:

CollegeVibes is an application created in my Introduction to Human Computer Interaction Class. It is angled to assist freshman commuter students make friends and meet like-minded people.



















The Process:







The Interview Stage [PDF]

For this stage, we had to think of questions to ask interviewees centeralizing on the routes they take to make friends. After interviewing 3 students at Montclair State University, we then proceeded to find patterns in their responses to the questions we asked.

After deciding on the target demographic:

First-year non-transfer commuters at Montclair State University.

We began creating personas.


  • Persona #1 [PDF]


  • Persona #2 [PDF]


  • Personal #3 [PDF]


Raw Materials [PDF]

The goal was to find 5 websites and/or applications and select a 'Big Idea' from each that we thought would be useful to solving our solution. In the PDF, there are slides dedicated to explaining how the 'Big Idea' worked in the application/website as well as Pros & Cons of applying the 'Big Idea' to our application.

Conveying the Solution [PDF]

We took our Persona #1, Mason Cooper, and developed a Context Scenerio. In the Context Scenerio, we explained a scenerio in which he would use the CollegeVibes application to make a friend. A storyboard is included as well for Persona #1.

Testing and Refining Our Solution [AXURE]

We used the program Axure to create a Key Path Scenerio angled specifically to an actual student at Montclair State University. After interviewing the student beforehand, we were able to create a version of CollegeVibes tailored to her interests. Below is a video of the student using the application.

CollegeVibes Demo - Usablity Test [YouTube Video]

In this video, a fellow teammate - Sarah Fathelbab - is using the Think Aloud Protocol. We test the main features of the CollegeVibes application and how the student interacted with the application as a whole.
